# nerve

## Purpose

`nerve/` is the bounded planning, analysis, and implementation-design cluster for the V Ecosystem runtime-support build now referred to as **Nerve**.

Nerve is the V Ecosystem's own runtime-support direction.
It is not claw-code.
External repos may be studied as pattern mines, but this directory is for shaping the V Ecosystem's governed Nerve build in shared-root terms.

`nerve/` is **not** an authority root.
It is **not** doctrine.
It is **not** a substitute for the canonical docs under `interfaces/`, `governance/`, `project-v/`, `veda/`, or `v-forge/`.

This directory exists to hold:

- Nerve analysis
- doctrinal audits
- source-mapped implementation references
- bounded planning artifacts
- superseded investigation material

Any conclusion here that affects active ecosystem behavior must be promoted into the proper canonical doc cluster elsewhere in the shared root.

## Folder rules

### `analysis/`
Exploratory and synthesis docs.
These are inputs, not decisions.

### `audits/`
Risk, compliance, and doctrinal-fit judgments.
These are evaluative docs, not implementation authority.

### `references/`
Code-grounded or source-mapped lookup material.
These should stay factual and implementation-facing.

### `planning/`
Bounded planning artifacts that decide next steps, sequencing, and adoption posture.
These bridge investigation work into real authority-doc changes.

### `archive/`
Superseded or no-longer-active material.
Use this aggressively to keep `nerve/` readable.

---

## Operating rule

Do not let `nerve/` become a shadow authority root.
If a conclusion becomes load-bearing, move it into the correct canonical doc cluster.

Do not treat external repos as blueprints.
Use them as bounded reference inputs while keeping Nerve as a V-native governed build.
