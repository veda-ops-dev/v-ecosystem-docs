# Desktop UX

## Purpose
Define how desktop UX material is organized under the shared V Ecosystem docs root.

## Scope
This directory holds desktop UX research inputs, working direction material, and factual reference notes that support the Tauri 2 desktop application direction.

## Out of Scope
This directory does NOT replace Tier 1 or Tier 2 authority.
It does NOT define governance doctrine.
It does NOT override canonical interface, lifecycle, approval, or system-boundary docs under `ecosystem/`, `governance/`, `interfaces/`, `project-v/`, `veda/`, or `v-forge/`.

## System
Cross-system desktop operator application for Project V, VEDA, and V Forge.

## Rules / Doctrine
- `ux/desktop/` is implementation-support material subordinate to shared-root authority.
- `research/` holds non-canonical research reports, audits, and external analysis inputs.
- `direction/` holds cleaned, repo-native desktop UX direction docs that may guide implementation but do not override canonical doctrine.
- `references/` holds factual source notes, pattern notes, and bounded implementation references.
- Research material must not masquerade as authority.
- If a conclusion here becomes load-bearing for ecosystem behavior, it must be promoted into the correct canonical doc cluster elsewhere in the shared root.
- Do not let `ux/desktop/` become a shadow authority root.

## Usage
Use `research/` for exploratory reports and architecture audits.
Use `direction/` for the endorsed desktop UX direction after it has been rewritten into repo-native form.
Use `references/` for supporting notes on external patterns or implementation observations.
